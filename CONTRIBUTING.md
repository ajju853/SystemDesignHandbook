# Contributing

Thank you for contributing to System Design Mastery!

## Guidelines
- Each topic follows the same format: Definition, Real Example, Advantages, Disadvantages, Diagram, Interview Questions
- Files should be concise and practical
- Diagrams can be ASCII art or Mermaid
- Link between related topics using relative paths
- Keep the learning path in mind — beginners should be able to follow sequentially

## Adding a Topic
1. Create a new markdown file in the appropriate module folder
2. Update the module's README.md to include the new topic
3. Follow the existing format

## Adding a Project
1. Create a new file in `12-Hands-On-Projects/`
2. Include: Requirements, Capacity Estimation, API Design, DB Design, HLD, LLD, Scaling, Deployment
